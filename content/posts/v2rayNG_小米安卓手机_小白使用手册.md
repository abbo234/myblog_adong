# v2rayNG 在小米安卓手机上的小白使用手册

> 适用对象：小米 / Redmi / POCO 安卓手机用户  
> 系统范围：HyperOS / MIUI 近年版本  
> 客户端：v2rayNG  
> 使用方式：第三方服务商订阅链接导入  
> 用户水平：小白  
> 核验日期：2026-06-12

---

## 0. 资料来源与时效性声明

本手册基于以下信息整理：

1. v2rayNG 官方 GitHub 仓库：`2dust/v2rayNG`。仓库说明显示 v2rayNG 是 Android 端 V2Ray 客户端，支持 Xray core 和 v2fly core。
2. v2rayNG GitHub Releases：当前核验到 Latest 版本为 `2.2.3`，发布日期为 2026-06-02。
3. v2rayNG 官方界面字符串：当前版本中存在 `Subscription group setting`、`Update subscription`、`Real delay config`、`Tcping config`、`Per-app proxy` 等入口。
4. 小米官方支持页：未知来源安装权限路径、后台自启动权限、后台权限相关说明。
5. 小米 / MIUI 后台机制参考资料：小米系统对后台应用限制较强，锁后台、自启动、电池不限制对 VPN 类应用稳定性有影响。

重要说明：

- 本手册不推荐任何服务商。
- 本手册不要求你提供完整订阅链接、二维码、账号密码、付款信息或后台完整截图。
- 小米手机型号很多，HyperOS / MIUI 菜单文字可能略有差异。找不到路径时，优先在系统设置顶部搜索关键词：`安装未知应用`、`后台自启动`、`电池`、`通知`、`VPN`。
- 如果 GitHub Release 和第三方下载页版本不一致，优先以 GitHub Release 为准。
- 如果你用的是非常旧的小米手机，APK 架构选择可能不同；现代小米 / Redmi / POCO 手机一般优先选择 `arm64-v8a`。

你需要准备：

## 2. 下载 v2rayNG

### 2.1 你要点哪里

优先从 v2rayNG 官方 GitHub Release 下载。

操作思路：

1. 打开浏览器。
2. 进入 v2rayNG 官方 GitHub 仓库。
3. 找到 `Releases`。
4. 进入标记为 `Latest` 的版本。
5. 找到 `Assets` 下载区。
6. 下载适合手机的 APK。

### 2.2 你要下载哪个文件

现代小米 / Redmi / POCO 安卓手机，一般优先找文件名同时包含：

- `.apk`
- `arm64-v8a`

例如文件名可能类似：

```text
v2rayNG_版本号_arm64-v8a.apk
```

注意：不要死记具体文件名。版本号会变，文件名也可能变化。重点看两个关键词：

```text
arm64-v8a + .apk
```

### 2.3 为什么选 arm64-v8a

你不用深入理解这个词。

只要记住：

- 现代安卓手机大多是 ARM 64 位架构。
- 小米 / Redmi / POCO 近年机型通常优先选 `arm64-v8a`。
- `x86`、`x86_64` 通常更偏模拟器或特殊设备。
- 如果你不确定，且 Release 页面提供 `universal` 版本，也可以选 `universal`。

### 2.4 成功标志

下载完成后，你能在以下位置看到 APK：

- 浏览器下载列表
- 通知栏下载完成提示
- 文件管理器 → 下载 / Download

### 2.5 失败先检查

1. 是不是从 GitHub Release 下载，不是从群文件、网盘、未知网站下载。
2. 文件名里是否有 `.apk`。
3. 现代小米手机是否优先选择了 `arm64-v8a`。
4. 浏览器是否拦截了 APK 下载。
5. 手机存储空间是否足够。

不建议安装服务商发来的未知修改版 APK。

---

## 3. 在小米手机上安装 APK

### 3.1 你要点哪里

1. 打开 **文件管理器**。
2. 进入 **下载 / Download** 文件夹。
3. 找到刚下载的 v2rayNG APK。
4. 点击 APK。
5. 点击 **安装**。

### 3.2 如果提示禁止安装未知来源应用

小米官方给出的常见路径之一是：

```text
设置 → 更多设置 → 隐私 → 特殊应用权限 → 安装未知应用
```

有些系统版本也可能是：

```text
设置 → 隐私保护 → 特殊权限 → 安装未知应用
```

你要做的是：

1. 找到你刚才用来打开 APK 的应用。
   - 如果你从浏览器下载并打开，就授权浏览器。
   - 如果你从文件管理器打开，就授权文件管理器。
2. 打开 **允许来自此来源**。
3. 返回 APK 安装页面。
4. 再次点击安装。

### 3.3 你要填什么

不需要填账号密码。

只需要给当前安装来源一个临时安装权限。

### 3.4 成功标志

安装完成后，桌面或应用列表里出现：

```text
v2rayNG
```

### 3.5 失败先检查

1. 是否授权了正确的应用：浏览器或文件管理器。
2. APK 是否下载完整。
3. 是否下载了错误架构的 APK。
4. 手机是否开启了更严格的安全拦截。
5. 存储空间是否不足。

安装完成后，建议关闭“允许安装未知应用”。

路径仍然是：

```text
设置 → 更多设置 / 隐私保护 → 特殊应用权限 → 安装未知应用
```

这样可以降低以后误装恶意 APK 的风险。

---

## 4. 首次打开 v2rayNG 与 VPN 权限

### 4.1 你要点哪里

1. 打开 v2rayNG。
2. 先不要乱改设置。
3. 等导入订阅、选择节点后，再点击连接按钮。
4. 第一次连接时，手机可能弹出：

```text
VPN 连接请求
```

你需要点击：

```text
确定 / 允许
```

### 4.2 这个 VPN 权限是什么意思

这不是让你重新买一个 VPN。

这是 Android 系统在问：

> 是否允许 v2rayNG 接管手机网络连接？

不允许的话，v2rayNG 无法正常工作。

### 4.3 成功标志

- v2rayNG 显示正在连接或已连接。
- 手机状态栏出现 VPN 图标。

### 4.4 失败先检查

1. 是否点了拒绝。
2. 是否已有其他 VPN 软件正在运行。
3. 小米安全中心是否拦截。
4. 是否没有选择节点就直接点连接。

注意：状态栏出现 VPN 图标，只代表手机网络流量正在被某个 VPN 类应用接管，不代表服务商一定可信。

---

## 5. 导入第三方订阅链接

这一节是核心。

当前 v2rayNG 版本中，稳定路径建议走：

```text
订阅分组设置 → 添加订阅分组 → 保存 → 回主界面更新订阅
```

不要只在主界面找 `+`。有些版本或界面布局里，主界面可能没有明显的 `+`。

---

### 5.1 先复制服务商订阅链接

#### 你要点哪里

1. 打开服务商后台。
2. 找到类似这些入口：
   - 订阅链接
   - 一键导入
   - v2rayNG 订阅
   - V2Ray 订阅
   - 通用订阅
3. 复制订阅链接。

#### 你要填什么

这一步不用填内容，只复制链接。

#### 成功标志

手机提示：

```text
已复制
```

或者你确认剪贴板里已经有订阅链接。

#### 失败先检查

1. 不要复制网页登录地址。
2. 不要复制 Clash 专用订阅，除非服务商明确说 v2rayNG 可用。
3. 不要复制单个节点链接当成订阅。
4. 不要把完整订阅链接发给别人。

---

### 5.2 进入订阅分组设置

#### 你要点哪里

1. 打开 v2rayNG。
2. 找到菜单入口。
   - 可能是右上角三个点。
   - 可能是左上角菜单。
   - 可能在设置页面里。
3. 找到：

```text
订阅分组设置
```

如果界面是英文，找：

```text
Subscription group setting
```

#### 你要填什么

还没填，先进入页面。

#### 成功标志

你进入一个管理订阅分组的页面。

页面里通常可以添加、编辑或删除订阅分组。

#### 失败先检查

1. 你是否打开的是 v2rayNG，不是其他客户端。
2. 是否进入了设置菜单。
3. 是否界面语言是英文，需要找 `Subscription group setting`。

---

### 5.3 添加订阅分组

#### 你要点哪里

在订阅分组设置里，点击：

```text
添加 / 新增 / Add
```

不同版本按钮位置可能不同。

#### 你要填什么

通常填两项：

1. **备注 / 名称 / remarks**

可以填：

```text
我的订阅
```

或：

```text
常用节点
```

2. **订阅地址 / URL / Optional URL**

粘贴服务商给你的完整订阅链接。

其他高级选项默认不动。

#### 成功标志

订阅分组设置页面里出现你刚添加的分组。

例如：

```text
我的订阅
常用节点
服务商订阅
```

#### 失败先检查

1. 是否把链接粘贴到了 URL / 订阅地址位置。
2. 是否忘记填写备注。
3. 是否忘记点击保存。
4. 订阅链接是否复制完整。

---

### 5.4 回主界面更新订阅

只添加订阅分组还不够。

你必须回主界面更新订阅，节点才会出现。

#### 你要点哪里

1. 保存订阅分组。
2. 返回 v2rayNG 主界面。
3. 打开菜单。
4. 点击：

```text
更新订阅
```

如果界面是英文，找：

```text
Update subscription
```

#### 你要填什么

不用填内容。

#### 成功标志

主界面出现节点列表。

例如：

```text
香港节点
日本节点
新加坡节点
美国节点
自动选择
```

具体名字由服务商决定。

#### 失败先检查

1. 手机普通网络是否能用。
2. 是否添加了订阅分组但忘记更新订阅。
3. 订阅链接是否过期。
4. 服务商后台套餐是否有效。
5. 是否复制成网页登录地址。
6. 服务商是否正在维护。

---

## 6. 扫码导入和剪贴板导入

### 6.1 扫码导入

适合情况：服务商后台给你的是二维码。

#### 你要点哪里

1. 打开 v2rayNG。
2. 找到导入入口。
3. 选择：

```text
扫描二维码导入
```

英文可能是：

```text
Import from QRcode
```

4. 对准服务商后台二维码。
5. 导入后保存。
6. 回主界面更新订阅或查看节点。

#### 成功标志

主界面出现节点，或者订阅分组里出现配置。

#### 失败先检查

1. 摄像头权限是否允许。
2. 二维码是否完整显示。
3. 二维码是否是 v2rayNG 支持的格式。
4. 是否把二维码截图泄露给别人。

不要把订阅二维码发给 AI 或群友。

---

### 6.2 从剪贴板导入

适合情况：你已经复制了链接。

#### 推荐路径

优先使用：

```text
订阅分组设置 → 添加订阅地址 → 保存 → 回主界面更新订阅
```

如果当前版本提供：

```text
从剪贴板导入
```

英文可能是：

```text
Import from Clipboard
```

也可以尝试。

#### 成功标志

- 节点出现在主界面；或
- 订阅分组被成功添加；或
- 更新订阅后出现节点列表。

#### 失败先检查

1. 剪贴板里是否是完整订阅链接。
2. 是否是服务商支持 v2rayNG 的订阅。
3. 是否导入后没有更新订阅。

---

## 7. 更新订阅

### 7.1 为什么要更新订阅

订阅链接只是一个入口。

更新订阅后，v2rayNG 才会去服务商那里拉取最新节点。

### 7.2 什么时候需要更新

- 第一次添加订阅后
- 节点突然全部超时
- 服务商通知线路调整
- 套餐刚续费
- 长时间没打开 v2rayNG
- 换手机后重新导入

### 7.3 你要点哪里

回到 v2rayNG 主界面，打开菜单，选择：

```text
更新订阅
```

英文界面找：

```text
Update subscription
```

### 7.4 成功标志

- 节点列表刷新。
- 节点数量变多或恢复正常。
- 没有明显报错。

### 7.5 失败先检查

1. 手机普通网络能不能打开网页。
2. 服务商套餐是否过期。
3. 订阅链接是否复制错。
4. 服务商是否在维护。
5. 是否使用了错误格式的订阅。

---

## 8. 选择节点与测速

### 8.1 你要点哪里

在主界面点一个节点。

然后打开菜单，找类似：

```text
测试全部配置真连接
测试延迟
Tcping
Real delay config
```

不同版本文字可能略有不同。

### 8.2 成功标志

节点旁边出现延迟数字。

例如：

```text
80 ms
150 ms
300 ms
```

### 8.3 怎么选节点

普通用户按这个顺序选：

1. 延迟较低。
2. 倍率正常。
3. 稳定。
4. 地区符合需求。
5. 不要迷信高价节点一定快。

### 8.4 注意

- 延迟低不等于下载速度一定快。
- 倍率高可能更耗流量。
- 某个地区节点不可用，不代表全部节点都坏了。
- 节点名好看不代表线路稳定。

## 9. 启动连接

### 9.1 你要点哪里

1. 在节点列表里点选一个节点。
2. 点击 v2rayNG 的连接按钮。
3. 第一次连接时，系统弹出 VPN 请求，点击允许。

### 9.2 你要填什么

不用填内容。

### 9.3 成功标志

至少满足以下几项：

- v2rayNG 显示已连接。
- 状态栏出现 VPN 图标。
- 当前节点延迟正常。
- 浏览器能打开目标网站。
- IP 检测结果显示出口地区变化。

### 9.4 失败先检查

1. 是否选中了节点。
2. 是否允许 VPN 权限。
3. 是否有其他 VPN 软件占用。
4. 节点是否超时。
5. 订阅是否过期。

---

## 10. 验证是否连接成功

不要只凭“网页能不能打开”判断。

按这个顺序检查：

1. v2rayNG 是否显示已连接。
2. 状态栏是否有 VPN 图标。
3. 当前节点是否有延迟。
4. 浏览器是否能访问目标网站。
5. IP 检测网站显示的出口地区是否变化。
6. 其他 App 是否能联网。
7. 日志是否有明显错误。

可能误判的情况：

- 浏览器缓存导致网页看起来能打开。
- DNS 异常导致部分网站打不开。
- 分应用代理设置错，导致只有部分 App 走代理。
- 节点地区不符合目标网站要求。

---

## 11. 断开连接与恢复普通网络

### 11.1 你要点哪里

1. 打开 v2rayNG。
2. 点击连接按钮，让它断开。
3. 观察状态栏 VPN 图标是否消失。

### 11.2 成功标志

- v2rayNG 显示未连接。
- 状态栏 VPN 图标消失。
- 手机恢复普通网络。

### 11.3 断开后仍不能上网，先检查

1. 开关飞行模式一次。
2. 切换 Wi-Fi / 移动流量。
3. 检查是否还有其他 VPN 软件在运行。
4. 重启手机。
5. 仍不行，再考虑重置网络设置。

不用时建议断开连接。

## 13. 后台自启动

### 13.1 什么时候需要

如果你希望 v2rayNG 更稳定，或者锁屏后不要轻易断开，可以打开后台自启动。

### 13.2 路径

小米官方支持页给出的常见路径：

```text
设置 → 应用 → 权限 → 后台自启动
```

也可能显示为：

```text
设置 → 应用 → 权限管理 → 自启动
```

### 13.3 操作

1. 打开设置。
2. 进入应用。
3. 找到权限。
4. 找到后台自启动。
5. 找到 v2rayNG。
6. 打开允许。

### 13.4 成功标志

v2rayNG 后面的开关处于开启状态。

### 13.5 代价

可能略微增加耗电。

## 14. 电池设置：设为不限制

### 14.1 为什么要设置

小米系统可能为了省电清理后台应用。

v2rayNG 被清理后，VPN 可能断开。

### 14.2 路径

常见路径：

```text
设置 → 应用 → 应用管理 → v2rayNG → 省电策略 / 电池
```

也可能是：

```text
设置 → 电池 → 应用电池管理 → v2rayNG
```

### 14.3 建议选择

选择类似：

```text
无限制
不限制
允许后台运行
关闭电池优化
```

不要选择：

```text
限制后台活动
省电限制
智能限制但频繁断开
```

### 14.4 成功标志

v2rayNG 的电池策略显示为“不限制”或类似状态。

### 14.5 代价

可能更耗电。

现实判断：如果你只是偶尔使用，不一定要一直开不限制；如果你需要锁屏不断线，就需要检查这一项。

## 15. 最近任务锁定

### 15.1 为什么要锁定

小米系统清理后台时，最近任务里被锁定的 App 更不容易被清掉。

### 15.2 操作

1. 打开 v2rayNG。
2. 回到桌面。
3. 打开最近任务界面。
4. 找到 v2rayNG 卡片。
5. 长按卡片，或点卡片附近的菜单。
6. 选择：

```text
锁定
```

或点击小锁图标。

### 15.3 成功标志

v2rayNG 卡片旁边出现小锁图标。

### 15.4 失败先检查

1. 当前系统版本是否隐藏了这个入口。
2. 是否需要长按卡片才出现菜单。
3. 是否被极致省电模式影响。

## 16. 通知权限

### 16.1 建议

建议允许 v2rayNG 通知。

原因：

- 你能看到连接状态。

- 断开时更容易发现。

### 16.2 路径

```text
设置 → 通知与状态栏 → 应用通知 → v2rayNG → 允许通知
```

或：

```text
设置 → 应用 → 应用管理 → v2rayNG → 通知管理
```

### 16.3 成功标志

v2rayNG 通知权限开启。

## 17. 锁屏后断开怎么办

按这个顺序查：

1. v2rayNG 是否仍在后台运行。
2. 是否开启后台自启动。
3. 电池策略是否设为不限制。
4. 最近任务是否锁定 v2rayNG。
5. 通知权限是否允许。
6. 是否开了省电模式 / 超级省电模式。
7. 是否 Wi-Fi 和移动流量切换导致断开。
8. 节点是否本身不稳定。

不要一上来重装。

重装可能导致配置丢失，问题反而更复杂。

# 第三部分：看不懂时再看概念

---

## 18. v2rayNG 是什么

v2rayNG 是安卓手机上的代理客户端。

它负责：

- 读取订阅配置
- 显示节点
- 建立连接
- 接管手机网络流量

它不负责：

- 出售节点
- 提供套餐
- 保证服务商可靠
- 帮你充值续费

一句话：

```text
v2rayNG 是工具，节点和订阅由服务商提供。
```

开源客户端可信，不等于服务商可信。

---

## 19. 订阅链接是什么

订阅链接是服务商给你的配置地址。

里面通常包含：

- 节点列表
- 套餐身份信息
- token
- 更新地址

泄露后，别人可能盗用你的套餐和流量。

所以：

```text
不要公开订阅链接。
不要公开订阅二维码。
不要把完整订阅链接发给 AI。
```

---

## 20. 订阅链接和单个节点链接的区别

### 20.1 订阅链接

特点：

- 包含一组节点。
- 可以更新。
- 服务商换节点后，你更新订阅就能拿到新节点。
- 推荐小白优先使用。

### 20.2 单个节点链接

特点：

- 只导入一个节点。
- 节点坏了就不能自动换。
- 不适合长期维护。

小白优先找：

```text
v2rayNG 订阅
V2Ray 订阅
通用订阅
订阅链接
```

不要优先复制单个节点。

---

## 21. 节点是什么

节点可以理解成网络出口服务器。

不同节点可能不同：

- 地区
- 延迟
- 速度
- 倍率
- 稳定性

节点慢或超时，不一定是 v2rayNG 坏了，也可能是服务商线路问题。

---

## 22. VPN 权限是什么

Android 的 VPN 权限表示：

```text
允许这个 App 接管手机网络连接。
```

这是一种系统权限。

它不代表：

- 服务商一定可信
- 线路一定安全
- 所有 App 一定都走代理

不用时建议断开。

---

## 23. 路由模式 / 代理模式

你可能看到这些模式：

- 全局代理
- 绕过局域网和大陆地址
- 规则分流
- 直连

小白建议：

- 默认不要乱改。
- 日常使用优先规则 / 分流类模式。
- 排错时可以短时间试全局代理。
- 用完及时断开。

不要一上来改复杂路由。

---

## 24. 分应用代理

分应用代理就是：

```text
只让某些 App 走代理，其他 App 不走代理。
```

普通用户默认不要开启。

只有在这些情况才考虑：

- 浏览器要走代理，其他 App 不走。
- 某个 App 必须走代理。
- 某些国内 App 不想走代理。

如果设置错，可能出现：

```text
浏览器能用，但其他 App 不能用。
```

---

## 25. DNS 设置

DNS 可以理解成：

```text
把网站名字翻译成服务器地址。
```

DNS 异常可能导致：

- v2rayNG 显示已连接，但网页打不开。
- 只有部分网站打不开。
- 某些 App 网络异常。

小白不要一上来改 DNS。

排错顺序应该是：

1. 换节点。
2. 更新订阅。
3. 切换 Wi-Fi / 移动流量。
4. 检查分应用代理。
5. 最后才考虑 DNS。

# 第四部分：常见故障排查

排错原则：

```text
先查简单原因，再查复杂原因。
先查手机网络，再查订阅，再查节点，再查设置，再考虑服务商问题。
不要一上来改 DNS、路由、高级设置。
不要一上来重装。
```

## 26. APK 无法安装

**现象：** 点 APK 后无法安装。

**可能原因：**

- 未允许安装未知应用。
- APK 下载损坏。
- 下载了错误版本。
- 小米安全扫描拦截。
- 存储空间不足。

**先检查：**

1. 是否授权浏览器或文件管理器安装未知应用。
2. 是否从 GitHub Release 下载。
3. 文件名是否包含 `.apk`。
4. 现代手机是否优先选 `arm64-v8a`。
5. 存储空间是否足够。

**解决方法：**

重新下载 APK，授权安装来源，再安装。

**成功标志：**

桌面出现 v2rayNG 图标。

---

## 27. 安装后打不开

**现象：** 打开 v2rayNG 闪退或没反应。

**可能原因：**

- APK 损坏。
- 版本不兼容。
- 手机系统限制。
- 安装包来源异常。

**先检查：**

1. 是否从官方 GitHub Release 下载。
2. 是否选错架构。
3. 是否安装了未知修改版。

**解决方法：**

卸载后重新从官方 Release 下载合适 APK。

**成功标志：**

能进入 v2rayNG 主界面。

---

## 28. 找不到订阅入口

**现象：** 主界面找不到 `+`，不知道在哪里导入订阅。

**可能原因：**

- 当前版本界面变化。
- 主路径不是 `+`。
- 需要进订阅分组设置。

**检查步骤：**

1. 打开 v2rayNG 菜单。
2. 找 `订阅分组设置`。
3. 英文界面找 `Subscription group setting`。
4. 进入后添加订阅地址。
5. 保存后回主界面更新订阅。

**成功标志：**

更新订阅后主界面出现节点列表。

---

## 29. 订阅导入失败

**现象：** 添加订阅后没有节点。

**可能原因：**

- 链接没复制完整。
- 复制成网页登录地址。
- 服务商套餐过期。
- 订阅不是 v2rayNG 支持格式。
- 添加后忘记更新订阅。

**检查步骤：**

1. 重新复制订阅链接。
2. 确认是 v2rayNG / V2Ray / 通用订阅。
3. 确认订阅分组已保存。
4. 回主界面更新订阅。
5. 查看服务商后台套餐状态。

**联系服务商时提供：**

- 打码后的错误提示
- 手机型号
- 系统版本
- v2rayNG 版本
- 不要提供完整订阅链接

---

## 30. 更新订阅失败

**现象：** 点更新订阅后报错。

**可能原因：**

- 手机普通网络异常。
- 订阅过期。
- 服务商后台故障。
- 订阅地址被复制错。

**先检查：**

1. 浏览器能不能打开普通网页。
2. 服务商后台能不能登录。
3. 套餐是否有效。
4. 订阅链接是否完整。

**解决方法：**

重新复制订阅链接，重新添加订阅分组，再更新订阅。

**成功标志：**

节点列表正常出现。

---

## 31. 节点全部超时

**现象：** 所有节点测速都是超时。

**可能原因：**

- 手机网络异常。
- 订阅过期。
- 服务商线路故障。
- 没有更新订阅。
- 当前网络环境限制。

**检查步骤：**

1. 先断开 v2rayNG，用普通网络打开网页。
2. 更新订阅。
3. 切换 Wi-Fi / 移动流量。
4. 查看服务商公告。
5. 联系服务商。

**不要先改：**

- DNS
- 路由规则
- TUN
- 高级参数

## 32. 点击连接后没反应

**现象：** 点连接按钮没有变化。

**可能原因：**

- 没选节点。
- 没允许 VPN 权限。
- 其他 VPN 正在运行。
- v2rayNG 被系统限制。

**先检查：**

1. 是否选中了节点。
2. 是否允许 VPN 请求。
3. 是否关闭其他 VPN 软件。
4. 是否开启后台权限。

**成功标志：**

状态栏出现 VPN 图标。

## 33. 显示已连接但打不开网页

**可能原因：**

- 节点不可用。
- DNS 异常。
- 路由规则异常。
- 分应用代理设置错。
- 浏览器缓存。
- 服务商线路问题。

**检查步骤：**

1. 换一个节点。
2. 断开后重新连接。
3. 切换 Wi-Fi / 移动流量。
4. 检查是否开启分应用代理。
5. 更新订阅。
6. 再考虑 DNS 或路由。

**成功标志：**

浏览器能打开目标网站，IP 检测地区发生变化。

## 34. 浏览器能用，其他 App 不能用

**可能原因：**

- 分应用代理排除了某些 App。
- App 自身网络权限关闭。
- 路由模式导致部分流量直连。
- DNS 异常。

**先检查：**

1. 是否开启分应用代理。
2. 目标 App 是否被排除。
3. 小米系统是否禁止该 App 使用网络。
4. 该 App 是否本身服务异常。

**解决方法：**

先关闭分应用代理，再测试。

## 35. 锁屏后断开

**优先检查小米系统设置：**

1. 后台自启动是否开启。
2. 电池策略是否“不限制”。

4. 通知权限是否允许。
5. 是否开启超级省电模式。
6. Wi-Fi 是否锁屏断开。

## 36. 切换 Wi-Fi / 移动网络后断开

**可能原因：**

- 网络切换导致 VPN 重连。
- 当前节点不稳定。
- 系统后台限制。

**先检查：**

1. 等 10-20 秒看是否自动恢复。
2. 手动断开再连接。
3. 换节点。
4. 检查后台和电池设置。

# 第五部分：安全提醒

---

## 39. 开源客户端不等于服务商可信

v2rayNG 可以是开源客户端。

但服务商控制的是：

- 节点
- 订阅
- 账号
- 流量出口
- 售后
- 日志策略

所以：

```text
客户端可信 ≠ 服务商可信
```

---

## 40. 服务商可能掌握什么

### 通常可以掌握

- 订阅账号
- 套餐状态
- 流量使用量
- 订阅是否过期
- 节点使用情况

### 可能掌握

- 连接时间
- 出口节点
- 部分连接统计
- 部分 DNS 行为

### 取决于服务商策略

- 是否记录域名
- 是否记录连接目标
- 日志保存多久
- 是否共享数据

不要夸大恐慌，也不要天真相信。

---

## 41. 不建议使用的服务商特征

看到这些信号要谨慎：

- 价格低到不合理
- 只在群聊售卖
- 没有稳定官网
- 频繁更换域名
- 强制安装未知客户端
- 不说明退款规则
- 不说明流量倍率
- 节点经常不可用
- 客服只催续费
- 要求不必要个人信息
- 付款方式异常
- 只给你不明 APK，不给官方客户端下载方式

现实判断：

```text
越不透明，用户承担的失败风险越高。
```

## 42. 排错时哪些信息能发，哪些不能发

| 信息类型 | 能不能发给 AI / 群友 / 客服 | 说明 |
|---|---:|---|
| 完整订阅链接 | 不能 | 可能包含 token |
| 订阅二维码 | 不能 | 等同于订阅链接 |
| 服务商后台完整截图 | 不建议 | 可能泄露账号、套餐、token |
| 打码后的错误提示 | 可以 | 保留错误文字，遮住链接和账号 |
| 手机型号 | 可以 | 例如 Redmi / Xiaomi / POCO 型号 |
| 系统版本 | 可以 | 例如 HyperOS / MIUI 版本 |
| v2rayNG 版本号 | 可以 | 用于判断兼容性 |
| 节点名称 | 谨慎 | 最好打码服务商特征 |
| 日志全文 | 谨慎 | 先检查是否含 token、IP、账号 |

## 43. 更稳妥的使用习惯

建议：

- 优先从官方 GitHub Release 下载 v2rayNG。
- 不安装服务商提供的未知修改版 APK。
- 不公开订阅链接。
- 不公开订阅二维码。
- 不在不可信代理环境下处理高敏感事务。
- 服务异常时先停止续费。
- 保留付款和沟通记录。
- 出现异常扣费、失联、无法退款，及时止损。
- 不用时主动断开连接。

21. 浏览器测试是否能访问目标网站。

23. 不用时断开连接。
24. 不要泄露订阅链接。
