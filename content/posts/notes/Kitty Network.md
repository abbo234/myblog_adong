# Kitty Network AIR 套餐使用手册

> 适用对象：已经购买 Kitty Network AIR 套餐的普通用户。  
> 适用场景：Windows 11、Android 手机，通过第三方订阅链接导入代理客户端进行普通网页浏览、AI 工具访问、资料查询、视频观看等合法用途。  
> 手册定位：使用说明 + 故障排查 + 风险控制。不是服务商宣传稿，也不保证服务长期稳定。

---

## 0. 先给结论

你购买的 **AIR 套餐**，定位应当是：

> **低成本、轻度使用、备用优先，不建议当唯一主力代理服务。**

AIR 的核心价值是价格低，不是线路一定高级、IP 一定干净、隐私一定强、访问一定稳定。

建议使用方式：

- 日常使用选择 **Rule / 规则模式**。
- 不要长期全局代理。
- 不要用于银行、政务、主邮箱、支付类账号等高敏感场景。
- 不要 BT/PT 下载，不要做攻击、扫描、批量注册、刷量、垃圾营销等高风险行为。
- 不要泄露订阅链接。
- 不要频繁重置订阅。
- 先稳定用满一个周期，再决定是否续费或升级。

## 1. AIR / STANDARD / PREMIUM 的区别

截至本手册编写时，Kitty Network 官方价格页展示的三档套餐大致为：

| 套餐 | 周期价格 | 月流量 | 现实定位 |
|---|---:|---:|---|
| AIR | 24 元 / 年 | 128G/月 | 轻度使用、备用、低成本试错 |
| STANDARD | 15 元 / 季 | 512G/月 | 普通日常使用，性价比更均衡 |
| PREMIUM | 9 元 / 月 | 1024G/月 | 重度使用，多设备、视频更多 |

重要判断：

- 套餐差异首先是 **流量和付款周期**。
- 不要默认 PREMIUM 就一定比 AIR 拥有更干净 IP 或更高级线路，除非服务商明确标注节点权限不同。
- AIR 的 128G/月足够普通网页、ChatGPT、GitHub、少量 YouTube，但不适合长期 4K 视频、大文件下载、多设备全天候挂代理。

## 2. 必须理解的几个概念

### 2.1 订阅链接

订阅链接不是普通网址，而是服务商给你的节点配置入口。客户端通过它拉取节点列表、分流规则、DNS 配置等信息。

风险点：

- 谁拿到你的订阅链接，谁就可能使用你的流量。
- 不要发到群里、论坛、截图、博客、GitHub、网盘公开链接。
- 如果怀疑泄露，应去官网重置订阅，但不要频繁重置。

### 2.2 节点

节点就是你选择的代理出口。不同节点可能有不同国家/地区、线路、协议、负载、IP 质量。

判断节点不能只看延迟。更重要的是：

- 能否打开目标网站；
- 晚高峰是否稳定；
- 是否经常超时；
- ChatGPT / Claude / YouTube 是否可用；
- 连续使用 30 分钟是否断流。

### 2.3 Rule / Global / Direct

| 模式 | 含义 | 建议 |
|---|---|---|
| Rule / 规则 | 国内直连，国外走代理 | 日常默认选择 |
| Global / 全局 | 所有流量都走代理 | 只用于临时测试 |
| Direct / 直连 | 所有流量都不走代理 | 用于排查问题 |

普通用户日常使用：**Rule 模式即可**。

### 2.4 系统代理

系统代理会让大多数浏览器和部分软件走代理，但不是所有应用都会听系统代理。

适合：

- Chrome / Edge 浏览器；
- Telegram Desktop；
- 一些支持系统代理的软件。

可能不生效：

- 部分游戏；
- 部分 UWP 应用；
- 部分自带网络栈的软件；
- 部分国内软件。

### 2.5 TUN 模式

TUN 模式类似虚拟网卡，可以接管更多系统流量。

普通用户判断：

- 浏览器能用：先不开 TUN。
- 某些软件不走代理：再考虑开 TUN。
- 开 TUN 后没网：立即关闭 TUN，回到系统代理模式排查。

TUN 更强，但也更容易与防火墙、杀毒软件、系统网络设置冲突。

### 2.6 DNS 泄漏

DNS 是把域名转换成 IP 的过程。代理开了，但 DNS 查询如果还走国内运营商，就可能出现 DNS 泄漏。

检测方法：

- 打开 DNS 泄漏检测网站；
- 看 DNS 服务器是否出现中国大陆运营商；
- 如果出现，说明 DNS 分流或代理 DNS 配置可能有问题。

普通用户不要一上来乱改系统 DNS。优先使用客户端订阅自带配置。

## 3. Windows 11：Clash Verge Rev 使用流程

### 3.1 下载与安装

推荐从 Clash Verge Rev 官方文档或 GitHub Release 下载。不要随便从网盘、群文件、陌生博客下载客户端。

Windows 11 普通电脑一般选择 x64 安装包。

安装后，首次启动可能会看到 Windows 防火墙提示。正常情况下允许即可。

### 3.2 导入 Kitty Network 订阅

操作步骤：

1. 登录 Kitty Network 官网面板。
2. 找到你的订阅链接。
3. 复制订阅链接。
4. 打开 Clash Verge Rev。
5. 进入 **订阅 / Profiles / 配置** 页面。
6. 点击 **新建 / 导入 / Import**。
7. 类型选择 **URL / 订阅链接**。
8. 名称填写：`Kitty Network AIR`。
9. URL 粘贴你的订阅链接。
10. 保存。
11. 点击更新订阅。

成功标志：

- 订阅列表出现 `Kitty Network AIR`。
- 节点列表能正常显示。
- 没有出现红色报错。

### 3.3 启用配置

导入后需要确认该订阅处于启用状态。

操作：

1. 在配置列表中选中 `Kitty Network AIR`。
2. 点击启用 / 使用。
3. 回到首页。
4. 出站模式选择 **Rule / 规则**。
5. 打开 **系统代理**。

成功标志：

- 首页显示系统代理已开启。
- 浏览器访问 Google / GitHub / ChatGPT 可打开。
- 国内网站如百度、淘宝仍能正常打开。

### 3.4 节点选择

建议顺序：

1. 先选自动选择 / Auto / URL-Test。
2. 如果目标网站打不开，再手动切换节点。
3. ChatGPT / Claude 优先测试美国、日本、新加坡等节点。
4. Telegram 慢时可测试美国或荷兰节点。
5. YouTube 慢时，优先测试距离近且负载低的节点。

不要只选延迟最低的节点。延迟低不代表速度快。

### 3.5 是否开启 TUN

默认建议：**先不开 TUN，只开系统代理。**

需要 TUN 的场景：

- 浏览器能用，但某个软件不能用；
- 某些应用不识别系统代理；
- 你明确需要让更多系统流量走代理。

不建议长期无脑开 TUN。开 TUN 后如果出现无法联网、DNS 异常、国内网站打不开，先关闭 TUN 再排查。

## 4. Android：v2rayNG 使用流程

适合对象：安卓手机用户，例如小米、OPPO、一加、vivo 等。

### 4.1 安装

推荐从 v2rayNG 官方 GitHub Release 下载 APK。不要安装来源不明的修改版。

安装后首次打开，系统可能提示 VPN 权限。连接节点时需要允许。

### 4.2 导入订阅

不同手机和 v2rayNG 版本界面略有差异。有些手机界面右上角不一定显示 `+`，可以从 **订阅分组设置** 导入。

通用流程：

1. 打开 v2rayNG。
2. 进入右上角菜单。
3. 找到 **订阅分组设置**。
4. 新增订阅分组。
5. 备注填写：`Kitty Network AIR`。
6. 地址粘贴 Kitty Network 订阅链接。
7. 保存。
8. 回到主界面。
9. 选择更新订阅。
10. 等待节点列表出现。

成功标志：

- 主界面出现多个节点。
- 点击节点后可以选择。
- 点击右下角连接按钮后，手机顶部出现 VPN 标志。

### 4.3 连接测试

连接后测试：

1. 打开浏览器。
2. 访问 Google。
3. 访问 YouTube。
4. 访问 ChatGPT。
5. 再打开百度确认国内网站是否正常。

如果 Google 能打开、百度也能打开，说明基础代理可用。

### 4.4 Android 常见问题

#### 问题 1：更新订阅失败

优先检查：

- 订阅链接是否复制完整；
- 手机时间是否准确；
- 当前网络是否能访问订阅域名；
- 是否有空格、换行、多余字符；
- 是否刚刚重置订阅但复制了旧链接。

#### 问题 2：节点导入了，但全部超时

可能原因：

- 订阅过期；
- DNS 解析异常；
- 本地网络不稳定；
- 客户端版本过旧；
- 当前地区或运营商连接该服务商节点不稳定。

处理顺序：

1. 更新订阅；
2. 换节点；
3. 换 WiFi / 移动数据测试；
4. 更新 v2rayNG；
5. 联系服务商。

#### 问题 3：能连上，但网页打不开

处理顺序：

1. 断开代理，看国内网站是否正常；
2. 换节点；
3. 重启浏览器；
4. 检查手机是否开启了私人 DNS；
5. 临时关闭省电模式、流量管控、防火墙类软件。

## 6. AIR 套餐的流量控制

AIR 每月 128G，不是无限流量。合理使用很关键。

### 6.1 大致流量消耗参考

| 使用行为 | 流量消耗倾向 | AIR 是否适合 |
|---|---|---|
| ChatGPT / Claude 文本对话 | 很低 | 适合 |
| Google / GitHub / 查资料 | 低 | 适合 |
| Telegram 文字聊天 | 低 | 适合 |
| YouTube 720p | 中等 | 可以 |
| YouTube 1080p | 中高 | 控制时长 |
| YouTube 4K | 高 | 不建议长期看 |
| 大文件下载 | 高 | 不建议 |
| BT/PT | 极高且违规 | 禁止 |

### 6.2 省流量建议

- 使用 Rule 模式。
- 国内网站不要走代理。
- 视频默认 720p 或 1080p，不要长期 4K。
- 不要开全局挂一天。
- 不要让云盘、系统更新、游戏平台走代理下载大文件。
- 电脑和手机不要同时长期挂代理。

### 6.3 什么时候需要升级 STANDARD

符合任一情况再考虑升级：

- 每月 128G 不够；
- 你每天看较多 YouTube；
- 电脑和手机都长期使用；
- 经常月底流量见底；
- STANDARD 明确提供更多可用节点，而不是只多流量。

如果只是查资料、用 AI、少量视频，AIR 继续用即可。

## 7. 节点质量测试方法

### 7.1 不要只看延迟

延迟只是参考。你真正要看的是可用性和稳定性。

测试顺序：

1. 更新订阅。
2. 选择 Rule 模式。
3. 测 Google。
4. 测 YouTube。
5. 测 ChatGPT。
6. 测 GitHub。
7. 晚高峰再测一次。
8. 连续使用 30 分钟，看是否掉线。

### 7.2 晚高峰测试

建议在 UTC+8 时间：

- 20:00–23:30；
- 或服务商文档中提到的 18:00–次日 01:00 高峰段。

测试内容：

- YouTube 1080p 是否卡顿；
- ChatGPT 是否频繁断；
- GitHub 是否能打开；
- Telegram 图片是否加载慢；
- 节点是否大量超时。

### 7.3 结果判断

| 现象 | 判断 |
|---|---|
| 白天可用，晚高峰明显变差 | 线路或负载问题 |
| 某个节点不可用，其他正常 | 单节点问题 |
| 所有节点超时 | 订阅、DNS、本地网络或服务商问题 |
| 浏览器能用，软件不能用 | 系统代理覆盖不足，考虑 TUN |
| 国内网站变慢 | 可能开了 Global 或规则异常 |
| ChatGPT 不可用，YouTube 可用 | IP 被目标平台限制，换节点 |

## 8. 常见故障排查

### 8.1 订阅更新失败

可能原因排序：

1. 订阅链接复制错误；
2. 订阅链接过期或被重置；
3. 当前网络访问订阅域名不稳定；
4. 客户端版本过旧；
5. 系统时间错误导致 TLS 校验失败；
6. DNS 解析异常；
7. 服务商订阅接口异常。

处理步骤：

1. 从官网重新复制订阅链接。
2. 确认链接没有空格和换行。
3. 检查系统时间是否自动同步。
4. 换 WiFi / 手机热点测试。
5. 换客户端测试。
6. 截图报错，联系服务商。

### 8.2 TLS 证书校验失败

这类错误通常发生在“更新订阅”或“连接节点”阶段。

优先检查：

- 系统时间是否正确；
- 订阅链接是否是最新；
- 浏览器打开订阅链接是否返回正常配置，而不是网页错误；
- 本地 DNS 是否解析异常；
- 客户端是否最新版；
- 杀毒软件或代理链是否拦截 HTTPS。

不要一上来关闭证书校验。关闭校验会降低安全性，不建议普通用户这样做。

### 8.3 No Proxy Group Found

可能原因：

- 订阅格式不是 Clash/Mihomo 配置；
- 订阅转换失败；
- 客户端不兼容；
- 配置中缺少 proxy-groups；
- 你拿到的是错误页面，不是真正订阅内容。

处理步骤：

1. 更新客户端。
2. 换 Clash Verge Rev / FlClash 测试。
3. 从官网复制 Clash/Mihomo 专用订阅。
4. 如果仍失败，发截图给服务商。

### 8.4 No Host Name Found

通常与 DNS 或节点域名解析有关。

处理步骤：

1. 更新订阅。
2. 换节点。
3. 换网络。
4. 检查系统 DNS。
5. 让客户端使用订阅默认 DNS。
6. 联系服务商确认节点域名是否正常。

### 8.5 节点全部超时

先不要急着判断跑路。按顺序排查：

1. 关闭代理，测试国内网站；
2. 更新订阅；
3. 换节点；
4. 换网络；
5. 检查客户端版本；
6. 检查 DNS；
7. 查看日志；
8. 联系服务商。

如果多个设备、多个网络、多个客户端都全部超时，才更可能是服务商侧问题。

### 8.6 开代理后全网没网

常见原因：

- 开了错误配置；
- TUN 冲突；
- DNS 配置异常；
- 防火墙拦截；
- 系统代理残留。

处理步骤：

1. 关闭 TUN。
2. 关闭系统代理。
3. 退出客户端。
4. 重启浏览器。
5. 重新打开客户端。
6. 只开启系统代理，不开 TUN。
7. Rule 模式测试。

### 8.7 浏览器能用，其他软件不能用

说明系统代理没有覆盖该软件。

处理方式：

- 软件内手动设置 HTTP/SOCKS 代理；
- 或开启 TUN 模式；
- 或换支持代理的软件版本。

普通用户先不要复杂配置，先确认浏览器可用。

## 9. 隐私与账号安全

### 9.1 机场能看到什么

一般代理服务商至少有能力看到：

- 你的真实连接 IP；
- 连接时间；
- 使用流量；
- 部分目标域名或连接方向；
- 订阅账号信息。

HTTPS 网站的具体内容通常被加密，服务商一般不能直接看到你的密码和网页具体内容。但这不等于完全隐私。

### 9.2 不建议做的事

不要用 Kitty Network 或任何普通机场做以下事情：

- 登录网银、政务、主邮箱、支付账号；
- 处理高度敏感个人资料；
- 长期绑定重要平台账号；
- 批量注册、刷量、垃圾营销；
- 爬取需要规避平台规则的数据；
- 违法下载、BT/PT、攻击扫描、撞库等行为。

### 9.3 账号风险判断

某个平台能不能用，不只取决于 IP。还取决于：

- 账号历史；
- 登录地区变化；
- 浏览器指纹；
- 设备环境；
- 行为频率；
- 平台自身风控策略。

不要把“原生 IP”“家宽 IP”“低风险值”当成不会风控的保证。

## 10. 服务条款与使用限制要点

根据你提供的 Kitty Network 文档，用户需要特别注意：

1. 存在无退款政策。
2. 服务商保留修改、暂停或终止服务的权利。
3. 服务不保证永久可用或不中断。
4. BT/PT、垃圾邮件、攻击、滥用等行为禁止。
5. 部分端口会被 WARP 中转或直接屏蔽。
6. 部分域名、服务、下载类型会被审计或阻断。
7. 访问内容、法律责任由用户自行承担。
8. 订阅信息不得公开传播。

现实含义：

## 12. 续费和升级判断

### 12.1 继续用 AIR 的条件

满足以下条件，可以继续 AIR：

- 每月 128G 够用；
- ChatGPT、Google、GitHub 可用；
- YouTube 1080p 偶尔能看；
- 晚高峰还能接受；
- 你只是轻度使用；
- 它只是备用，不是唯一主力。

### 12.2 升级 STANDARD 的条件

满足以下情况，考虑 STANDARD：

- AIR 流量不够；
- 每月经常接近 128G；
- 你每天都看视频；
- 多设备使用频繁；
- STANDARD 确认比 AIR 多节点或更稳定。

### 12.3 不建议继续投入的情况

出现以下情况，不建议续费或升级：

- 订阅经常无法更新；
- 节点大面积不可用；
- 晚高峰基本不能用；
- 客服无法有效处理；
- 官网、面板、订阅域名频繁异常；
- 你需要它承担工作刚需或重要账号环境。
